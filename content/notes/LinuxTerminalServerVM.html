How to Create a Working Linux terminal Server using Oracle VirtualBox

An useful guide for peeps who want to host game servers (e.g., Minecraft, Terraria, AzerothCore, etc.) or other applications on a Linux Terminal Server.

Prerequisites

  • Oracle VirtualBox: virtualbox.org
  • Debian ISO: mirror.cogentco.com

    • ⚠️"Click, download, and/or execute links or applications at your own risk."⚠️

    1. VirtualBox Installation & VM Creation

    1. Install VirtualBox:
      • Download and install VirtualBox like any other application. Ensure you select the desired options during installation.

    2. Download Debian ISO:
      • Download the Debian ISO and place it in an accessible folder.

    3. Create a New Virtual Machine:
      • Open VirtualBox and click "New" to create a new VM.
      • Name: Give your VM a name (e.g., "DebianVM").
      • Folder: Select the folder where the VM will be stored.
      • ISO Image: Select the Debian ISO you downloaded. This will automatically disable further selection on this tab.
      • Type: Select "Linux".
      • Version: Select "Debian (64-bit)".
      • Skip Unattended Installation: Ensure you skip unattended installation to manually configure the OS.

    4. Configure Hardware Settings:
      • Memory: Allocate at least 4-8GB of RAM.
      • Processors: Assign 4-12 CPUs depending on your system's capabilities.

    5. Virtual Hard Disk:
      • Allocate sufficient storage for your server. A minimum of 32GB is recommended, but adjust based on your needs.

    2. Installing the OS

    1. Start the VM:
      • After creating the VM, select it and click "Start".
      • The VM will boot into the Debian installer.

    2. BIOS Install Screen:
      • Select "Install" (not "Graphical Install") since this is a server VM.

    3. System Configuration:
      • Language: Select your system language and locale.
      • Keyboard: Choose your keyboard layout.

    4. Hostname:
      • Set a hostname for your VM. This will be the name used to identify the machine on the network.

    5. Domain Name:
      • If you don’t know what to enter, use the default value provided.

    6. Root Account:
      • Set a strong password for the root account (the administrative account).

    7. User Account:
      • Create a regular user account:
        • Real Name: Enter your name or an alias.
        • Username: Choose a username for login.
        • Password: Set a strong password for the user account.

    8. Partition Disks:
      • Use "Guided - use entire disk" (the default option) if you’re unsure.
      • Proceed with partitioning the disk.

    9. Install Base System:
      • The base system installation will begin now. This may take some time.

    10. Configure Package Manager:
      • When prompted to select a Network Mirror, choose No and select the region closest to you.
      • This step is crucial for installing additional packages later.

    11. Software Selection:
      • Only select "SSH Server" and "Standard System Utilities" for a minimal server setup.

    12. GRUB Installation:
      • Install GRUB to the master boot record (MBR) when prompted.
      • Select the default directory if one is provided.

    13. Finish Installation:
      • Once the installation is complete, click "Continue" to reboot the VM.

    3. Post-Installation

    1. Login:
      • After rebooting, you’ll be prompted to log in.
      • Use either the root account or the user account you created.

    2. Enjoy:
      • Your Linux Terminal Server is now ready for use! You can start configuring it for your specific needs (e.g., hosting game servers).

    Notes

  • Ensure your VM has sufficient resources (RAM, CPU, and storage) for the applications you plan to run.
  • Regularly update your system using "sudo apt update && sudo apt upgrade" to keep it secure and up-to-date.
